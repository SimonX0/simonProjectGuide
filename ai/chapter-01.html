1.1.2 大语言模型是什么？ ​

大语言模型（Large Language Model，LLM） 是一种基于深度学习的 AI 模型，通过在海量文本数据上进行训练，学会了理解和生成人类语言。

核心特点：

  1. 大规模训练数据

    • 使用互联网上的海量文本（书籍、文章、代码等）
    • 例如：GPT-4 使用了万亿级别的词汇量进行训练

  2. 超大参数规模

    • 模型参数量从几十亿到数万亿
    • 更多参数 = 更强的表达能力

  3. 涌现能力（Emergent Abilities）

    • 当模型规模达到一定程度时，突然出现训练时未明确教授的能力
    • 例如：推理、编程、数学计算等

1.2 LLM 的工作原理（简化版） ​

1.2.1 本质：超级预测机器 ​

大语言模型的核心工作原理可以用一句话概括：根据上下文预测下一个词。

输入："今天天气真"
LLM预测：下一个词是"好"的概率最大
输出："今天天气真好"

输入："今天天气真"
继续预测：接下来是"啊"、"呀"的概率较高
输出："今天天气真好啊"

1.2.2 训练过程 ​

第一阶段：预训练（Pre-training）
  目标：学习世界知识
  数据：万亿级别的文本
  结果：一个会"续写文本"的基础模型

第二阶段：指令微调（Instruction Tuning）
  目标：学会听从指令
  数据：问答对、对话数据
  结果：一个能理解任务的模型

第三阶段：人类反馈强化学习（RLHF）
  目标：对齐人类价值观
  数据：人类排序和反馈
  结果：有用、诚实、无害的AI助手

1.2.3 LLM 的局限 ​

虽然 LLM 非常强大，但也要了解它的局限性：

局限性说明应对策略
幻觉问题可能编造错误信息验证重要信息，使用 RAG 技术
知识截止训练后的事件不知道提供最新上下文，联网搜索
数学计算复杂计算易出错使用计算器工具
上下文限制记忆容量有限分段处理，使用 Memory
推理深度复杂推理可能失败拆解问题，Chain of Thought

1.4.3 实现多轮对话 ​

创建 examples/02_chat.py：

python
import os
from openai import OpenAI
from config import Config

client = OpenAI(api_key=Config.OPENAI_API_KEY)

def chat_bot():
    """简单的对话机器人"""
    print("🤖 简单对话机器人（输入'quit'退出）\n")

    # 对话历史
    messages = [
        {"role": "system", "content": "你是一个友好的AI助手，名字叫小徐。"}
    ]

    while True:
        # 获取用户输入
        user_input = input("\n你：").strip()

        # 退出条件
        if user_input.lower() in ['quit', 'exit', '退出']:
            print("小徐：再见！👋")
            break

        if not user_input:
            continue

        # 添加用户消息到历史
        messages.append({"role": "user", "content": user_input})

        try:
            # 调用API
            response = client.chat.completions.create(
                model=Config.MODEL_NAME,
                messages=messages,
                temperature=Config.TEMPERATURE,
                max_tokens=Config.MAX_TOKENS
            )

            # 获取AI回复
            ai_reply = response.choices[0].message.content

            # 添加AI回复到历史
            messages.append({"role": "assistant", "content": ai_reply})

            # 打印回复
            print(f"小徐：{ai_reply}")

        except Exception as e:
            print(f"❌ 错误：{e}")

if __name__ == "__main__":
    chat_bot()

1.6.1 流式输出（实时显示） ​

长文本生成时，流式输出可以提升用户体验：

python
def stream_chat():
    """流式输出示例"""
    print("AI：", end="", flush=True)

    stream = client.chat.completions.create(
        model=Config.MODEL_NAME,
        messages=[
            {"role": "user", "content": "写一个关于AI的小故事"}
        ],
        stream=True  # 启用流式输出
    )

    for chunk in stream:
        if chunk.choices[0].delta.content is not None:
            content = chunk.choices[0].delta.content
            print(content, end="", flush=True)

    print()  # 换行

if __name__ == "__main__":
    stream_chat()

python
from openai import OpenAI, APIError, RateLimitError, APIConnectionError

def safe_chat(user_input):
    """带有错误处理的聊天"""
    try:
        response = client.chat.completions.create(
            model=Config.MODEL_NAME,
            messages=[{"role": "user", "content": user_input}]

    except RateLimitError:
        return "❌ 请求太快，请稍后再试"

    except APIConnectionError:
        return "❌ 网络连接失败，请检查网络"

    except Exception as e:
        return f"❌ 未知错误：{e}"

def chat_with_usage(user_input):
    """显示Token使用情况"""
    response = client.chat.completions.create(
        model=Config.MODEL_NAME,
        messages=[{"role": "user", "content": user_input}]
    )

    # 获取使用情况
    usage = response.usage

    print(f"💬 回复：{response.choices[0].message.content}\n")
    print(f"📊 Token使用统计：")
    print(f"   输入：{usage.prompt_tokens} tokens")
    print(f"   输出：{usage.completion_tokens} tokens")
    print(f"   总计：{usage.total_tokens} tokens")

    # 粗略估算成本（GPT-3.5价格）
    cost_input = usage.prompt_tokens * 0.0005 / 1000  # $0.0005 per 1K tokens
    cost_output = usage.completion_tokens * 0.0015 / 1000
    total_cost = cost_input + cost_output
    print(f"   预估成本：${total_cost:.4f}")

Q4: 调用API时出现"RateLimitError"错误？

  1. 降低请求频率，添加延迟

  • LLM本质是概率模型，输出具有随机性
  • 即使temperature=0，也可能有微小变化
  • 解决方法：设置temperature=0获得更稳定的输出

1.9.3 成本和性能问题 ​

Q7: 如何降低API调用成本？

A: 成本优化技巧：

  1. 使用gpt-3.5-turbo而不是gpt-4
  2. 限制max_tokens避免过长输出
  3. 精简提示词，减少输入tokens
  4. 缓存常见问题的答案
  5. 使用temperature=0减少重试次数

Q8: API响应很慢怎么办？

A: 优化建议：

  1. 使用流式输出（stream=True）
  2. 检查网络连接
  3. 减少提示词长度
  4. 考虑使用更快的模型（gpt-3.5-turbo）

1.9.4 模型能力问题 ​

Q9: AI给出的代码有错误怎么办？

A: 这是常见现象，LLM可能产生错误代码：

  1. 要求AI解释代码逻辑
  2. 使用更低的temperature（0.1-0.3）
  3. 测试代码并让AI调试
  4. 要求AI提供单元测试

Q10: AI"幻觉"问题，编造不存在的信息？

A: 这是LLM的固有限制，应对方法：

  1. 总是验证重要信息
  2. 使用RAG技术提供准确上下文
  3. 提示AI"如果不确定，请直接说不知道"
  4. 使用最新的模型（幻觉会减少）
