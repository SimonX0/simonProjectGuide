Git基础入门 ​

什么是 Git？ ​

一句话解释：Git 是一个"时光机"，可以让你随时回到项目的任何历史版本。

想象一个场景：

没有 Git 的时候 ​

你在写毕业论文，为了保险起见：

毕业论文.doc
毕业论文_修改版.doc
毕业论文_最终版.doc
毕业论文_绝对不改版.doc
毕业论文_真的最后一版.doc
毕业论文_打死不改版.doc
毕业论文_再改是狗.doc

问题：

  • ❌ 文件一大堆，分不清哪个是最新的
  • ❌ 改错了想恢复，找不到之前的版本
  • ❌ 想知道改了什么内容，对比起来很麻烦

有了 Git 之后 ​

毕业论文.doc (Git 自动记录所有历史)

查看历史：
版本3 - 2024-01-15 14:30 - 添加第三章
版本2 - 2024-01-14 10:20 - 修改第二章

  • ✅ 只有一个文件，自动记录历史
  • ✅ 随时回到任何版本
  • ✅ 清楚看到每次改了什么

Git vs 其他工具 ​

传统方式（像公司的中央服务器） ​

     ┌─────────┐
     │ 服务器  │ ← 所有文件都在这里
     └─────────┘
        ↑   ↑
        │   │
     你的电脑  同事的电脑

问题：
❌ 服务器坏了，所有文件丢失
❌ 必须联网才能工作
❌ 服务器挂了，大家都不能干活

Git 方式（每个人都有完整备份） ​

┌──────────────┐

┌──────────────┐

✅ 每个人都有完整备份
✅ 可以离线工作（坐飞机也能写代码）

核心概念（用生活中的例子） ​

1. 仓库（Repository） ​

仓库 = 存放项目文件的文件夹

就像你的"工作目录"，Git 会在旁边记小本本，记录这个文件夹里所有变化。

bash
# 创建新仓库
git init
# 意思：Git 帮我看着这个文件夹！

# 复制别人的仓库
git clone https://github.com/用户名/项目.git
# 意思：把 GitHub 上的项目完整复制一份到我的电脑

2. 工作区、暂存区、本地仓库、远程仓库 ​

这个是最容易晕的！用餐厅来比喻：

┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  工作区   │ -> │  暂存区   │ -> │ 本地仓库  │ -> │ 远程仓库  │
│ (餐桌)   │add │  (点餐台) │commit│(厨房)    │push │(外卖)    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘

你在餐桌     你点了菜       厨师做好了    送外卖
上吃饭       告诉服务员     存在冰箱里    送到客户那

实际操作：

bash
# 1. 在工作区编辑文件（在餐桌上写作业）
echo "Hello Git" > test.txt

# 2. 添加到暂存区（把作业放到老师讲台上）
git add test.txt

# 3. 提交到本地仓库（老师把作业收进文件夹）
git commit -m "添加测试文件"

# 4. 推送到远程仓库（把文件夹寄给家长）
git push origin main

为什么这么麻烦？一步到位不行吗？

想象你写作业：

  • 工作区：你在草稿纸上演算（随便画，错了就扔）
  • 暂存区：你挑出觉得对的题目（准备上交）
  • 本地仓库：老师收作业，保存在档案袋
  • 远程仓库：学校把成绩上传到教育局系统

这样可以让你有"后悔药"！

3. 提交（Commit） ​

提交 = 拍照留念

每次提交就像给项目拍一张照片，记录当前的样子。

bash
git commit -m "添加了登录功能"
# 意思：给当前状态拍张照，备注"添加了登录功能"
1
2

为什么要写提交信息？

就像照片要写日期和说明：

  • "2024年春节全家福"
  • "小明第一次学会骑车"

这样才能知道每个版本是什么！

4. 分支（Branch） ​

分支 = 平行宇宙

想象你在玩游戏：

主线剧情 (main 分支)
  ├── 存档A：选择救公主 (feature 分支1)
  └── 存档B：选择当海盗 (feature 分支2)
1
2
3

你可以在存档A里尝试救公主，失败了就读档回到存档B当海盗，互不影响！

实际用途：

main (主分支 - 正式版本)
  ├── dev (开发分支 - 测试新功能)
  └── feature-login (功能分支 - 开发登录功能)

安装 Git ​

Windows 用户 ​

  1. 打开浏览器，访问：https://git-scm.com/download/win
  2. 点击 "Download for Windows"
  3. 双击下载的安装包
  4. 一路点击 "Next"（使用默认设置就行）
  5. 安装完成后，打开命令行（CMD 或 PowerShell）
bash
# 验证是否安装成功
git --version
# 输出：git version 2.43.0.windows.1
# 看到版本号就说明安装成功了！

配置 Git（就像办银行卡） ​

安装完成后，需要告诉 Git 你是谁：

bash
# 设置你的名字
git config --global user.name "张三"

# 设置你的邮箱
git config --global user.email "zhangsan@gmail.com"

# 查看配置
git config --list

这样团队协作时，出了问题知道找谁！

第一次使用 Git（手把手教学） ​

场景1：创建新项目 ​

bash
# 1. 创建项目文件夹
mkdir my-first-project
cd my-first-project

# 2. 初始化 Git 仓库
git init
# 输出：Initialized empty Git repository
# 意思：Git 开始看管这个文件夹了

# 3. 创建一个文件
echo "# 我的第一个Git项目" > README.md

# 4. 查看状态（Git 告诉你发生了什么）
git status
# 会看到：
# Untracked files: README.md
# 意思：这个文件 Git 还没开始跟踪

# 5. 添加文件（告诉 Git 跟踪这个文件）
git add README.md

# 6. 提交（拍照留念）
git commit -m "第一次提交：创建README文件"

# 7. 查看历史（看看到底拍了哪些照片）
git log

常见问题（新手必看） ​

Q1: Git 一定要用黑框框（命令行）吗？ ​

A: 不一定！

虽然命令行最强大，但如果你不习惯，可以用图形界面工具：

推荐工具（适合新手）：

  • GitHub Desktop - 简单直观
  • Sourcetree - 功能强大
  • VSCode - 编辑器自带 Git（推荐！）

但建议：还是要学一些基本命令，因为：

  • 命令行更快
  • 所有教程都用命令行
  • 遇到问题时，命令行更容易找答案

Q2: Git 和 GitHub 有什么区别？ ​

A: 经常混淆，但其实很简单：

GitGitHub
是什么软件（工具）网站（服务）
干什么管理版本存放代码
在哪里安装在你电脑上在浏览器访问
需要联网不需要需要

比喻：

  • Git = 你的相册管理软件
  • GitHub = 网盘相册（在线备份）

你可以只使用 Git（不联网），也可以配合 GitHub（联网备份）。

Q3: 必须要学所有命令吗？太难了！ ​

A: 完全不需要！

好消息：Git 有几百个命令，但你只需要掌握 10 个就能应付 90% 的情况！

必学命令（后面章节会详细讲）：

bash
git status      # 查看状态
git add .       # 添加文件
git commit      # 提交
git push        # 推送
git pull        # 拉取
git log         # 查看历史
git branch      # 查看分支
git checkout    # 切换分支
git merge       # 合并

Q4: 我怕改错了代码怎么办？ ​

A: 这就是 Git 最大的作用！

有 Git 的安全网：

  • ✅ 改错了？git restore 恢复
  • ✅ 提交错了？git reset 回退
  • ✅ 想看历史？git log 查看
  • ✅ 想回到过去？git reset 回到任意版本

大胆改，Git 会保护你！

进阶练习 ​

挑战：体验版本回退

bash
# 1. 再做一次修改
echo "## 更新日志" >> README.md
git add .
git commit -m "添加更新日志"

# 2. 查看所有提交
git log --oneline

# 3. 回到第一个版本（用实际的commit hash替换）
git log --oneline  # 找到第一个提交的hash，比如 abc1234
git reset --hard abc1234

# 4. 查看README.md的内容（应该回到最初版本）
cat README.md

# ⚠️ 注意：reset --hard会丢弃之后的修改，谨慎使用！

💡 学习提示 ​

  1. 不要害怕命令行：Git命令大多很短，多练习几次就熟悉了
  2. 理解后再操作：每个命令要知道它做什么，不要盲目执行
  3. 多做实验：在练习项目中随意尝试，出错也没关系
  4. 利用可视化工具：VSCode的Git界面很直观，可以帮助理解
  5. 记住常用命令：status、add、commit、log是最常用的
