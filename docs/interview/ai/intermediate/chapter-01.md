# Prompt工程基础面试题

## 核心概念

### 什么是Prompt工程？

Prompt工程是设计和优化输入给大语言模型的提示词，以获得更准确、相关和高质量输出的技术和艺术。

**核心要素**：
1. **清晰性**：指令明确，无歧义
2. **上下文**：提供足够的背景信息
3. **格式**：指定期望的输出格式
4. **示例**：提供少样本示例

### Prompt设计的核心原则？

**1. 清晰具体**

```
❌ 差：写一个函数
✅ 好：用JavaScript写一个异步函数，使用fetch API获取数据
```

**2. 提供上下文**

```
❌ 差：优化这段代码
✅ 好：这是一个处理百万级数据的Vue3组件，请优化其性能
```

**3. 指定格式**

```
❌ 差：列出优缺点
✅ 好：请以Markdown表格形式列出优缺点，包含具体数值对比
```

**4. 给出示例**

```
❌ 差：转换以下数据格式
✅ 好：
输入：{ name: "Alice", age: 25 }
输出格式：Alice is 25 years old
请按此格式转换：{ name: "Bob", age: 30 }
```

## 高级技巧

### Prompt Chaining

将复杂任务拆分成多个步骤，每个步骤使用独立的Prompt。

```
步骤1：提取文章关键词
步骤2：基于关键词生成摘要
步骤3：将摘要翻译成多语言
```

### 情境注入

```
背景：这是一个面向新手的Vue3教程
任务：解释computed和watch的区别
要求：使用简单的比喻，避免技术术语
```

### 迭代优化

```
第一版：给出初版答案
反馈：太复杂了，简化代码
第二版：给出优化后的答案
反馈：添加注释说明
最终版：完整的注释版本
```

## 上下文管理

### Token限制处理

**问题**：上下文超出模型Token限制

**解决方案**：
1. **摘要压缩**：将长文本压缩为摘要
2. **滑动窗口**：保留最近N条消息
3. **向量检索**：只检索相关片段
4. **分层处理**：先处理局部，再汇总

### 长文本处理

```
方案1：分段处理
- 将长文档分成段落
- 每段独立分析
- 最后合并结果

方案2：RAG检索
- 将文档切片存入向量库
- 根据问题检索相关片段
- 只将相关片段放入Prompt
```

```javascript

temperature: 0.6

// 高温度（0.8-1.0）：更创造性，适合头脑风暴

## 常见问题

### 如何让AI输出代码？

```
请使用Vue3和TypeScript实现以下功能：
- 使用组合式API
- 添加完整的类型注解
- 包含错误处理
- 添加注释说明关键逻辑
```

### 如何让AI遵循格式？

```
请严格按照以下JSON格式输出，不要添加任何其他内容：
{
  "title": "string",
  "tags": ["string"],
  "content": "string"
}
```

**方法2：使用思维链引导思考**
**方法3：给出示例期望输出**
**方法4：迭代式提问和修正**
